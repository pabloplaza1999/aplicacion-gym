Paso 1 — Validación de valor (sin implementación)

Usando el contexto disponible del proyecto, evalúa únicamente el valor de negocio y el impacto funcional de la solicitud actual.

Instrucciones
NO proponer código.
NO proponer implementación técnica.
NO modificar documentación.
Solo análisis funcional y de negocio.
Analiza estos 5 ejes
1. Valor para el usuario

¿Qué problema resuelve?

¿Qué mejora para el usuario final, operador o negocio?

2. Valor para el producto

¿La funcionalidad fortalece el producto?

¿Cubre una necesidad existente o habilita capacidades futuras?

3. Impacto operativo

¿Reduce trabajo manual?

¿Reduce errores?

¿Mejora visibilidad, control o trazabilidad?

4. Limitaciones

¿Qué NO resuelve?

¿Qué supuestos o dependencias podrían afectar el resultado esperado?

5. Relación valor vs complejidad

Clasificar:

Alto valor / Baja complejidad
Alto valor / Alta complejidad
Bajo valor / Baja complejidad
Bajo valor / Alta complejidad
Cierre

Incluir:

Beneficios esperados
Riesgos principales
Recomendación final

Terminar con:

Recomiendo [proceder / posponer / reformular] porque...

Esperar aprobación explícita antes de continuar al Paso 2.
